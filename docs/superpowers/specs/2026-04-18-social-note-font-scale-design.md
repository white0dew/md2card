# 社交图文字号缩放设计

## 目标

为 `社交图文` 主题新增可持久化的字号缩放参数，支持两种作用范围：

- 仅正文缩放
- 整体排版缩放

默认保持当前视觉尺寸不变，并确保导出 PNG 时正文、标题、列表和分页内容不会因新参数出现明显缺失或截断。

## 范围

- 仅作用于 `社交图文` 主题
- 仅新增一个字号倍数配置和一个作用范围配置
- 复用现有设置侧栏、Zustand persist 和 `SocialNoteCard` 渲染链路
- 不扩展为全局字号系统
- 不新增“标题单独字号”和“正文单独字号”两套同时生效的复杂配置

## 方案

### 数据层

在 `settings-store` 中新增两个字段：

- `socialFontScaleMode`
- `socialFontScale`

其中：

- `socialFontScaleMode` 取值为 `body` 或 `all`
- `socialFontScale` 为数字倍数，默认值 `1`

两个字段都纳入现有 Zustand persist 持久化与 migrate 修正逻辑。

### 交互层

在 `SettingsSidebar` 的社交图文设置区新增一组“字号缩放”配置，放在“字体风格”附近，沿用现有表单样式：

1. 一个作用范围选择器：
   - 仅正文
   - 整体
2. 一个字号倍数滑杆：
   - 最小值 `0.85`
   - 最大值 `1.3`
   - 步进 `0.05`
3. 一个实时文本标签：
   - 显示为 `0.85x`、`1.00x`、`1.15x` 这类格式

这样用户只需要理解一件事：当前倍数应用到哪个范围。

### 渲染层

`SocialNoteCard` 新增两个 CSS 变量：

- `--social-font-scale`
- `--social-body-font-size`

基础规则：

- 默认正文基准字号维持当前 `17px`
- 当模式为 `body` 时，仅 `.card-content` 的基准字号按倍数变化，标题、章节标题、引用、表格和代码块继续沿用当前固定尺寸
- 当模式为 `all` 时，社交标题、正文、各级标题、引用、列表、表格、代码块、时间文字等现有 `font-size` 均按同一倍数缩放

实现方式采用“先提取具名字号常量，再通过 `calc()` 乘倍数”的形式，避免散落魔法数字，并让测试能够稳定断言。

### 分页与导出

字号放大会直接影响分页高度，因此不单独引入新的分页算法，而是继续依赖现有真实 DOM 测量逻辑。验证以真实浏览器导出 PNG 为准，不以预览肉眼判断代替。

## 备选方案

### 方案 A：两个独立缩放参数

- 一个控制正文
- 一个控制整体
- 优点：自由度最高
- 缺点：叠加后难以理解和预测，状态复杂度不必要增加

### 方案 B：作用范围 + 倍数

- 一个模式字段决定作用范围
- 一个倍数字段控制强度
- 优点：交互清晰、实现集中、和现有配置模式一致
- 缺点：不支持同时把标题和正文调到两个不同节奏

### 方案 C：固定档位

- 只提供“小 / 默认 / 大 / 特大”
- 优点：实现最省
- 缺点：精度不足，用户很难微调到满意尺寸

本次选择方案 B。

## 错误处理

- 如果持久化值缺失或非法，`socialFontScaleMode` 回退到 `body`
- 如果持久化倍数缺失、不是数字或超出范围，`socialFontScale` 回退到 `1`
- UI 展示的值始终使用 clamp 后的合法结果

## 测试策略

先补失败测试，再补实现：

1. `settings-store` 暴露并持久化 `socialFontScaleMode`、`socialFontScale` 及对应 setter
2. `SettingsSidebar` 暴露“字号缩放”作用范围和倍数控件
3. `SocialNoteCard` 读取新增状态，并根据模式输出对应 CSS 变量 / `calc()` 字号规则
4. 真实浏览器导出 `社交图文` PNG，对照左侧 Markdown 与导出图片，确认无文字缺失、截断、段落丢失或分页不连续

## 风险与约束

- `all` 模式会同时放大标题和小字信息，分页波动比 `body` 模式更明显
- 过大的倍数可能导致导出页数显著增加，因此范围应严格限制在 `0.85 ~ 1.3`
- 必须复用现有真实导出流程验证，不能只看页面预览
