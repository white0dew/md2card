# 社交图文配色设计

## 目标

为 `社交图文` 主题新增两个可持久化的预设色板配置：

1. 背景颜色
2. 重点字体颜色

重点字体颜色仅作用于标题、章节标题、加粗和斜体文本，不影响普通正文、时间文字和引用层级。

## 范围

- 仅作用于 `社交图文` 主题
- 仅提供预设色块点选
- 不提供自定义 HEX / RGB 输入
- 不扩展为全局主题 token 系统

## 方案

### 数据层

在 `settings-store` 中新增两个字段：

- `socialBackgroundColor`
- `socialAccentColor`

两个字段都走现有的 Zustand persist 持久化，默认值保持当前主题视觉不变。

### 配置层

新增一个小型颜色预设模块，集中维护：

- 社交图文背景色调色板
- 社交图文重点色调色板
- 默认背景色
- 默认重点色
- 色块展示名称

这样可以避免在卡片组件和设置面板里散落硬编码颜色值。

### UI 层

在 `SettingsSidebar` 的社交图文设置区增加两组调色板：

- 背景颜色
- 重点字体颜色

每组使用一排可点击色块展示，当前选中项需要有明显边框和选中标记。

### 渲染层

`SocialNoteCard` 读取两个配置并将其映射到样式：

- 卡片背景色使用 `socialBackgroundColor`
- 标题、`h2-h6`、`strong`、`em` 使用 `socialAccentColor`

其他文案仍保持原有灰阶层级，避免整个主题被强调色淹没。

## 测试策略

- 补 source-inspection 测试，确认 store 暴露新字段和 setter
- 确认设置面板存在两组调色板入口
- 确认 `SocialNoteCard` 会读取并应用这两个配置

## 风险与约束

- 颜色需要控制数量，避免设置区过长
- 强调色不能过浅，否则导出 OCR 和屏幕阅读都容易失真
- 默认值必须与现有主题一致，避免老用户升级后视觉突变
