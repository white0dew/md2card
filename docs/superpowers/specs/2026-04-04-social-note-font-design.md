# 社交图文字体切换设计

## 目标

为 `社交图文` 主题新增可持久化的字体风格切换，并同步将默认正文字号调大一号。

## 范围

- 仅作用于 `社交图文` 主题
- 仅提供预设字体风格选择
- 不提供自定义字体栈输入
- 不扩展为全局字体系统

## 方案

### 数据层

在 `settings-store` 中新增一个字段：

- `socialFontPreset`

该字段沿用现有 Zustand persist 持久化，默认值保持当前宋体风格。

### 配置层

新增一个轻量字体预设模块，集中维护：

- 社交图文字体预设列表
- 默认字体预设
- 字体标签文案
- 预设值合法性校验

首版提供三种预设：

1. 宋体风：保持当前整体气质，作为默认值
2. 黑体风：用于更现代、更利落的排版
3. 圆体风：用于更轻松、更亲和的内容表达

### UI 层

在 `SettingsSidebar` 的社交图文设置区新增一个“字体风格”选择器，放在颜色配置附近，沿用现有表单视觉样式。

### 渲染层

`SocialNoteCard` 读取 `socialFontPreset`，将所选字体栈映射为卡片的主字体。标题、正文、章节标题、加粗和斜体统一使用同一套字体体系，不拆分标题字体和正文字体配置。

同时将默认正文的基础字号从 `16px` 调整到 `17px`，正文段落继承该字号，保持当前行高节奏不变，避免分页行为出现过大波动。

## 备选方案

### 方案 A：仅做整卡字体预设切换

- 优点：实现最小，导出稳定，和现有颜色配置模式一致
- 缺点：灵活性有限

### 方案 B：分开控制标题字体与正文字体

- 优点：视觉调节空间更大
- 缺点：配置复杂度上升，当前主题收益不足

### 方案 C：开放自定义字体输入

- 优点：自由度最高
- 缺点：浏览器字体缺失、导出一致性和兼容性风险都更高

本次选择方案 A。

## 测试策略

- 先补失败测试，确认 store 暴露字体字段和 setter
- 确认设置面板存在“字体风格”入口
- 确认 `SocialNoteCard` 会读取并应用字体配置
- 确认默认正文基础字号已从 `16px` 调整到 `17px`

## 风险与约束

- 预设字体必须以常见系统字体为主，降低缺字和环境差异风险
- 字号调整会轻微影响分页，必须做真实浏览器导出验证
- 默认字体仍需保持现有社交图文气质，避免老用户升级后视觉跳变过大
